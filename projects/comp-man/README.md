# comp-man

Per-project state for the `comp-man` project. The pipeline writes
into this folder; everything except `common-config.json` and this
`README.md` is generated by the workflows.

## Layout

```
projects/comp-man/
├── common-config.json              # company / project / state backend coords
├── README.md                       # this file
└── <catalog>-<service_name>/       # one folder per requested catalog instance
    │                                  e.g. cat-aws-vpc-demo, cat-aws-eks-platform
    ├── main.tf, variables.tf, …    # catalog code, copied from
    │                                  template/catalogs/<catalog>/<version>/
    ├── .catalog-version            # which catalog@version was rendered
    ├── backend.hcl                 # generated at provision time
    └── configs/
        ├── config.json             # base template from the catalog
        ├── config-nonprod.json     # rendered per-env tfvars
        └── config-prod.json
```

One slot per `(catalog, service_name)` pair. To spin up a second
instance of the same catalog, request it with a different
`service_name`.

## How it gets here

1. **`project-init`** workflow created this folder + `common-config.json`.
2. **`infra-request`** workflow stages a slot when you pick a
   `(catalog, version, service_name)` — opens a PR adding the slot
   dir. Per-env config files (`config-<env>.json`) are generated at
   request time from the catalog's `config.json` template, with
   `environment` + tag overlay applied.
3. **`infra-provision`** workflow runs terraform against an existing
   slot — picks an env, plans, pauses for approval, applies.

## Bootstrap (one-time, after the project-init PR merges)

```bash
make bootstrap PROJECT=comp-man
```

That reads `common-config.json` and creates:
- S3 bucket `comp-man-tfstate` (versioned, encrypted, public-access blocked)
- DynamoDB table `comp-man-tfstate-lock` (PAY_PER_REQUEST)

Re-running is safe — both checks skip if the resources exist.

See the top-level `SETUP.md` and `template/docs/01_AWS_GITHUB_ACCESS.md` for
one-time AWS/GitHub setup.
